# 基于AI的农业植保机器人监控软件系统

## 1. 项目概述

### 1.1 项目背景
随着现代农业技术的发展，农业植保机器人已成为精准农业的重要组成部分。本系统旨在构建一个基于AI的农业植保机器人监控软件平台，实现对机器人的远程监控、智能控制、数据分析与决策支持。

### 1.2 系统目标
1. 实现对多个农业植保机器人的集中监控与管理
2. 集成AI视觉识别技术，自动识别作物状态、病虫害情况
3. 提供远程控制功能，支持手动和自动作业模式
4. 实时采集环境数据与机器人状态数据
5. 生成统计分析报告，辅助农业生产决策
6. 建立完善的用户权限管理与操作日志系统

### 1.3 技术特点
- 基于Django框架的Web后端系统
- 前后端分离架构，RESTful API设计
- YOLOv8深度学习模型用于农业病害识别
- 实时视频流传输与处理
- 多机器人协同管理与调度
- 数据可视化与智能分析

## 2. 系统架构设计

### 2.1 整体架构
系统采用分层架构设计，主要包括：
- **表示层**: Web前端界面，基于HTML/CSS/JavaScript实现
- **业务逻辑层**: Django后端应用，处理业务逻辑
- **数据访问层**: Django ORM，管理数据库访问
- **数据存储层**: SQLite数据库（开发环境）/PostgreSQL（生产环境）
- **AI服务层**: YOLOv8深度学习模型服务
- **设备通信层**: WebSocket/HTTP通信，与机器人设备交互

### 2.2 技术栈
- **后端框架**: Django 4.x
- **前端技术**: HTML5, CSS3, JavaScript, Bootstrap
- **数据库**: SQLite（开发）, PostgreSQL（生产）
- **AI框架**: PyTorch, Ultralytics YOLOv8
- **通信协议**: WebSocket, HTTP RESTful API
- **数据可视化**: Chart.js, ECharts
- **部署环境**: Nginx, Gunicorn, Docker

### 2.3 数据库核心表设计

#### 2.3.1 用户管理模块
1. **User (用户表)**
   - 扩展Django AbstractUser，存储系统用户信息
   - 关键字段: `username`, `password`, `email`, `role`, `phone`, `avatar`
   - 角色区分: 管理员、操作员、查看者

2. **LoginLog (登录日志表)**
   - 记录用户登录历史
   - 关键字段: `user`, `login_time`, `ip_address`, `user_agent`, `status`

#### 2.3.2 设备管理模块
3. **RobotDevice (机器人设备表)**
   - 存储植保机器人基本信息与实时状态
   - 关键字段: `device_id`, `name`, `model`, `status`, `battery_level`, `location_lat`, `location_lng`
   - 状态枚举: 在线、离线、工作中、充电中、故障

4. **DeviceOperationLog (设备操作日志表)**
   - 记录对机器人的所有操作
   - 关键字段: `device`, `operator`, `operation_type`, `details`, `created_at`
   - 操作类型: 启动、停止、暂停、恢复、配置修改、遥控操作

#### 2.3.3 通信模块
5. **RobotConnection (机器人连接表)**
   - 管理机器人网络连接状态
   - 关键字段: `device`, `ip_address`, `port`, `is_connected`, `connected_at`
   - 一对一关联RobotDevice

6. **MessageLog (消息日志表)**
   - 记录与机器人的通信消息
   - 关键字段: `connection`, `direction`, `message_type`, `content`, `timestamp`
   - 消息方向: 接收(inbound)、发送(outbound)

#### 2.3.4 控制模块
7. **ControlCommand (控制命令表)**
   - 存储发送给机器人的控制命令
   - 关键字段: `device`, `command_type`, `parameters`, `status`, `sender`, `sent_at`
   - 命令类型: 前进、后退、左转、右转、停止、开始喷洒、停止喷洒等
   - 状态: 等待中、已发送、执行中、已完成、失败

8. **ManualControl (手动控制状态表)**
   - 存储手动控制状态信息
   - 关键字段: `device`, `is_active`, `joystick_x`, `joystick_y`, `spray_enabled`
   - 一对一关联RobotDevice，支持实时遥控

#### 2.3.5 AI识别模块
9. **AIModel (AI模型表)**
   - 存储训练的AI模型信息
   - 关键字段: `name`, `model_type`, `model_file`, `model_path`, `num_classes`, `class_names`
   - 模型类型: YOLOv8系列、YOLO11系列、自定义模型

10. **TrainingTask (训练任务表)**
    - 记录AI模型训练任务
    - 关键字段: `name`, `model_type`, `dataset_path`, `epochs`, `batch_size`, `status`, `progress`
    - 状态: 等待中、训练中、已暂停、已完成、失败

11. **InferenceRecord (推理记录表)**
    - 记录AI推理任务执行情况
    - 关键字段: `device`, `model`, `mode`, `source_file`, `result_file`, `confidence_threshold`, `total_detections`
    - 推理模式: 单张图片、视频文件、实时摄像头

12. **RecognitionTask (识别任务表)**
    - 存储作物识别任务
    - 关键字段: `device`, `task_type`, `status`, `image`, `result_json`, `confidence`
    - 任务类型: 作物与杂草识别、病虫害检测、成熟度判断

13. **CropWeedDetection (作物杂草检测结果表)**
    - 存储作物杂草识别详细结果
    - 关键字段: `task`, `crop_count`, `weed_count`, `density_map`, `detection_boxes`

14. **PestDiseaseDetection (病虫害检测结果表)**
    - 存储病虫害识别详细结果
    - 关键字段: `task`, `disease_type`, `disease_confidence`, `pest_type`, `pest_confidence`

15. **MaturityDetection (成熟度检测结果表)**
    - 存储作物成熟度识别结果
    - 关键字段: `task`, `maturity_level`, `maturity_percentage`, `fruit_count`

#### 2.3.6 监控模块
16. **SensorData (传感器数据表)**
    - 存储机器人采集的环境数据
    - 关键字段: `device`, `temperature`, `humidity`, `light_intensity`, `soil_moisture`, `latitude`, `longitude`
    - 实时记录机器人位置与环境参数

17. **VideoStream (视频流表)**
    - 管理机器人视频流信息
    - 关键字段: `device`, `stream_url`, `status`, `started_at`, `last_frame_time`
    - 一对一关联RobotDevice

18. **GPSLocation (GPS位置表)**
    - 记录机器人GPS轨迹
    - 关键字段: `device`, `latitude`, `longitude`, `altitude`, `accuracy`, `recorded_at`

#### 2.3.7 数据分析模块
19. **WorkSession (作业会话表)**
    - 记录机器人作业会话
    - 关键字段: `device`, `operator`, `start_time`, `end_time`, `duration`, `total_distance`, `area_covered`
    - 用于统计分析作业效率

20. **StatisticalReport (统计报告表)**
    - 生成各类统计报告
    - 关键字段: `device`, `report_type`, `start_date`, `end_date`, `total_work_hours`, `total_area`, `total_pesticide`
    - 报告类型: 日报、周报、月报、年报

21. **DashboardMetrics (大屏指标表)**
    - 存储系统关键指标
    - 关键字段: `date`, `total_devices`, `online_devices`, `total_work_area`, `total_work_hours`, `pest_alerts`
    - 支持数据可视化展示

#### 2.3.8 配置管理
22. **RecognitionConfig (识别配置表)**
    - 存储AI识别相关配置
    - 关键字段: `name`, `model_path`, `confidence_threshold`, `is_active`

### 2.4 E-R关系设计

#### 2.4.1 核心实体关系
1. **用户-设备关系**
   - 一个用户可以拥有多个机器人设备（一对多）
   - 一个设备只能属于一个用户（多对一）

2. **设备-连接关系**
   - 一个设备对应一个连接状态（一对一）
   - 一个连接包含多条消息记录（一对多）

3. **设备-控制关系**
   - 一个设备可以接收多条控制命令（一对多）
   - 一个设备对应一个手动控制状态（一对一）

4. **设备-监控关系**
   - 一个设备产生多条传感器数据记录（一对多）
   - 一个设备对应一个视频流配置（一对一）
   - 一个设备产生多条GPS位置记录（一对多）

5. **AI模型-任务关系**
   - 一个AI模型可以用于多个推理任务（一对多）
   - 一个训练任务生成一个AI模型（一对一/多对一）

6. **识别任务-结果关系**
   - 一个识别任务对应一个详细结果（一对一）
   - 结果分为三类：作物杂草、病虫害、成熟度

#### 2.4.2 主要外键关系
- `RobotDevice.owner` → `User.id`
- `RobotConnection.device` → `RobotDevice.id`
- `ControlCommand.device` → `RobotDevice.id`
- `SensorData.device` → `RobotDevice.id`
- `RecognitionTask.device` → `RobotDevice.id`
- `InferenceRecord.model` → `AIModel.id`
- `TrainingTask.trained_model` → `AIModel.id`
- `CropWeedDetection.task` → `RecognitionTask.id`

#### 2.4.3 数据流设计
- **实时数据流**: 机器人设备 → 传感器数据 → 监控界面
- **控制数据流**: 用户界面 → 控制命令 → 机器人设备
- **AI处理流**: 图像采集 → 推理任务 → 识别结果 → 分析报告
- **统计流**: 作业数据 → 统计聚合 → 可视化报表

   - 支持多机器人协同管理
   - AI模型可替换升级，支持新算法集成

3. **数据处理能力**
   - 数据库设计支持大数据量存储
   - 实时数据处理满足监控需求
   - 异步任务处理支持AI模型训练

   - 操作日志记录，确保操作可追溯

3. **易用的操作流程**
   - 简化的机器人控制界面
   - 一键式AI分析功能
   - 自动化报告生成

#### 2.5.4 风险分析与应对
1. **技术风险**
   - **风险**: AI识别准确率受环境因素影响
   - **应对**: 多模型融合，人工复核机制
   
2. **通信风险**
   - **风险**: 网络不稳定导致控制延迟
   - **应对**: 本地缓存，断线重连机制，心跳检测
   
3. **安全风险**
   - **风险**: 远程控制可能被非法入侵
   - **应对**: 多重身份验证，操作审计，数据加密

## 3. 功能模块详细说明

### 3.1 用户管理模块
- **用户注册与登录**: 支持多种角色注册，安全认证
- **权限管理**: 基于角色的权限控制（RBAC）
- **操作审计**: 完整的登录与操作日志记录

### 3.2 设备管理模块
- **设备注册**: 新设备接入与配置
- **状态监控**: 实时显示设备状态、电量、位置
- **设备配置**: 喷洒速率、巡检间隔等参数设置

### 3.3 远程控制模块
- **手动控制**: 虚拟摇杆控制机器人移动
- **自动控制**: 预设路径规划与自动作业
- **喷洒控制**: 精准农药喷洒控制

### 3.4 AI识别模块
- **实时识别**: 基于摄像头的实时病害识别
- **图像分析**: 上传图片进行离线分析
- **模型训练**: 自定义数据集训练新模型
- **结果可视化**: 检测结果标注与统计

### 3.5 数据监控模块
- **环境数据**: 温湿度、光照、土壤墒情监测
- **位置跟踪**: 实时GPS轨迹显示
- **视频监控**: 实时视频流查看与录制

### 3.6 数据分析模块
- **作业统计**: 作业面积、时长、农药用量分析
- **健康评估**: 作物健康度评分与趋势分析
- **报告生成**: 自动化日报、周报、月报
